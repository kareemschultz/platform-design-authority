# Prototype API surfaces

This page is a **boundary overview**, not a generated canonical operation reference. The canonical Draft contract remains `openapi/first-slice-v1.yaml`, with endpoint authority in `registry/endpoint-permissions.json`. The runtime implements the reviewed WS1 surface and the merged WS2 Catalog Product and Inventory ledger/workflow subset at the evidence revision. It does not implement every operation in the first-slice OpenAPI contract.

## Base URL

The local server listens at `http://localhost:3000` by default.

| Surface | Path | Authentication | Purpose |
|---|---|---|---|
| Process health | `GET /` | None | Returns plain-text `OK` |
| Better Auth | `GET|POST /api/auth/*` | Endpoint-dependent | Email/password and session endpoints |
| oRPC transport | `/rpc/*` | Procedure-dependent | Type-safe application procedures |
| oRPC OpenAPI handler | `/api-reference/*` | Procedure-dependent | Runtime-derived reference for composed procedures; not the canonical first-slice contract |

The server allows `GET`, `POST`, and `OPTIONS` from the exact configured `CORS_ORIGIN`. Credentials are enabled for the approved browser origin.

## Implemented procedure families

| Family | Implemented scope | Authority boundary |
|---|---|---|
| `healthCheck` | Public process-health response | No business authority |
| `privateData` | Authenticated scaffold response | Authentication only; returned user is not Party, membership, role, permission, or entitlement |
| `identity` | Current identity and active-context selection | Current session and active membership are revalidated |
| `organizations` | List/get/update organizations and list locations | `platform.organization.read` or `platform.organization.update` |
| `users` | List, invite, and suspend memberships | `platform.user.read`, `.invite`, or `.suspend` |
| `parties` | List/get/create/update Party records and create identity links | Party permissions; Better Auth identity remains separate |
| `roles` | List roles and assign a role | `platform.role.read` or `platform.role.assign` |
| `entitlements` | List tenant entitlements | `platform.entitlement.read`; entitlement is not actor permission |
| `sessions` | List and revoke the current account's sessions | Authenticated account ownership, with idempotency on revocation |
| `audit` | List tenant audit evidence | `platform.audit.read` and active tenant context |
| `catalog.products` | Create, get, list, update, activate, and archive Product aggregates | Catalog Product permissions, version preconditions, idempotency for commands |
| `inventory.adjustments` | Create, get, list, approve, and reverse adjustments | Inventory permissions; maker-checker and linked reversal rules |
| `inventory.counts` | Create, get, list, submit, and approve blind counts | Inventory count permissions and current version |
| `inventory.balances` | List derived stock balances | `inventory.balance.read`; projection is not the ledger authority |
| `inventory.transfers` | Create, get, list, dispatch, and receive transfers | Transfer permissions, conservation, partial-receipt, and version rules |

The exact runtime composition is `apps/server/src/router.ts`. Its tests prove selected transport mappings and denial behavior. The generated contract package contains the broader Draft first-slice operation catalog and must not be mistaken for evidence that every operation is bound to an application implementation.

## Typed client use

Web clients consume the generated `AppRouterClient` through the repository oRPC/TanStack Query adapter. Do not duplicate request or response shapes manually. New procedures must declare ownership, authentication, tenant context, permission or explicit authenticated context, validation, error behavior, idempotency where applicable, audit, and tests.

## OpenAPI authority

There are three distinct artifacts:

1. The runtime-derived oRPC reference reflects the procedures composed at the evidence revision.
2. Root `openapi/first-slice-v1.yaml` is the governed Draft contract for the planned vertical slice.
3. `packages/contracts/platform-api/src/generated.ts` is generated from governed endpoint metadata and supplies typed contract definitions; generation does not prove runtime binding.

None of these artifacts proves production readiness. Generated reference pages must identify their source revision and must not become a separately edited contract.

## Error and security behavior

- Runtime application failures map to typed problem details with a safe code, correlation ID, retryability, next action, and uncertainty marker where applicable.
- Authentication cookies use hardened attributes according to environment.
- Trusted origins must be exact URL origins.
- Protected WS1/WS2 procedures revalidate active context and permissions at the server boundary; commands use idempotency keys and version preconditions where their contract requires them.
- Production rate limits, full RLS topology, public support guarantees, and complete production error/telemetry evidence remain unresolved.

Do not expose the prototype server to the public internet or use real tenant data.

## Verification

```bash
curl --fail http://localhost:3000/
bun run test
bun run check-types
```

The CI workflow also builds the Bun server, starts the Node fallback entry, and probes the root health endpoint.
