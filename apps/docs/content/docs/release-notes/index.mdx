# Controlled platform prototype — 2026-07-12

This is an implementation-evidence checkpoint, not a customer release or production-readiness declaration.

## WS2 controlled-prototype closeout candidate — 2026-07-16

- The Catalog, Inventory ledger and workflows, durable internal event delivery and projections, bounded import, Strict Online Numbering, and Product/Inventory web experience are assembled at the controlled-prototype depth defined by `PDA-RDM-009`.
- The closeout change verifies and reconciles evidence; it does not turn this checkpoint into a customer release, pilot approval, supported upgrade, or production-readiness declaration.
- WS2 closeout remains conditional on the PR7 evidence gates, exact-head CI, independent concurrence, merge, and exact-`main` verification. This note does not claim those pending steps have occurred.
- RR-007 production PostgreSQL RLS role topology and RR-009 external assistive-technology, penetration-testing, and production-content evidence remain open. WS5 owns broad offline transport; pilot, production, founder, provider, legal, security, capacity, recovery, and operational gates remain outside this checkpoint.
- Any target not executed with an exact environment, sample, result, and limitation remains an open target rather than implied evidence.

## WS2 Product and Inventory web checkpoint — 2026-07-16

- Added generated-client Product list, exact SKU/barcode/state filters, create/edit, activation, and archive workflows.
- Added location-scoped balance projections with source, timestamp, reconciliation, and cursor evidence.
- Added governed Inventory adjustment, blind-count, transfer, and bounded Product/opening-stock import workflows.
- Added keyboard/mobile browser checks and stable Product/Inventory help ID `PDA-DOC-001`; formal assistive-technology and production accessibility conformance remain open.
- Kept browser authentication and oRPC requests on same-origin `/api/auth` and `/rpc` paths while using the private `PLATFORM_API_INTERNAL_URL` only for server-side proxying and rendering.
- Permission, entitlement, active context, version, idempotency, owner state, and tenant scope remain server-authoritative. This checkpoint does not close WS2, RR-007, RR-009, pilot, or production gates.

## Included

- Better-T-Stack-derived Bun monorepo with governed deviations recorded in `docs/implementation/SCAFFOLD_README.md`.
- Next.js web application with API connectivity status, email/password sign-up and sign-in, protected dashboard, theme selection, and sign-out.
- Hono server with public health, Better Auth routing, oRPC transport, OpenAPI handler, and Node fallback entry.
- Better Auth backed by Drizzle and PostgreSQL for authentication and sessions only.
- Tenant, organization, membership, Party linkage, business roles, permissions, entitlements, active context, session revocation, and allowlisted Audit evidence at the bounded WS1 controlled-prototype depth.
- Catalog, Inventory ledger/workflows, durable internal event delivery/projections, bounded import, and Strict Online Numbering at the separately evidenced WS2 controlled-prototype depth.
- Expo native scaffold with production native-auth exchange deferred.
- Shared platform-owned shadcn-derived UI source using the governed Base UI/Rhea configuration.
- Fumadocs portal with local search, LLM-readable routes, and repository-owned content.
- PostgreSQL 18.4 Compose baseline with `pg_stat_statements` as the only preloaded optional extension.
- Frozen install, formatting, typecheck, workspace and browser tests, production builds, Node health, registry freshness, prohibited-surface scan, dependency audit, PostgreSQL verification, and Docker image checks in CI.

- production tenant-isolation roles/RLS, support access, accessibility conformance, penetration testing, backup/restore exercises, capacity/SLO evidence, or operational certification;

## Compatibility notes

- Bun is the preferred prototype runtime; the Hono server retains a Node 24 fallback.
- oRPC remains pinned to the verified published package set recorded in the implementation conflict log.
- Deep Windows checkout paths can exceed Turbopack's path limit during the Fumadocs build; use a shorter local path or the Linux CI result.
- PostgreSQL extensions remain governed by ADR-0024 and are absent unless specifically allowed.

## Upgrade and support status

No compatibility or support window is promised for this prototype checkpoint. Dependency changes require current technology evidence, a reviewed diff, frozen-lock validation, runtime/build tests, and updated lessons. Customer-facing release notes begin only after an approved product release exists.
