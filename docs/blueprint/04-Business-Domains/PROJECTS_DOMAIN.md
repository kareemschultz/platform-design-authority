# Projects Domain

## Purpose

Own project planning, work breakdown, tasks, milestones, resources, time, costs, billing inputs, collaboration, risk, and delivery performance.

## Core Capabilities

- Projects, programs, portfolios, phases, milestones, and work breakdown structures
- Tasks, dependencies, assignments, checklists, and recurring work
- Project templates, methodologies, status, health, and governance gates
- Resource demand, allocation, capacity, skills, and availability
- Time, expense, material, equipment, and subcontractor capture
- Budgets, estimates, forecasts, commitments, actuals, and variance
- Fixed-price, time-and-materials, milestone, retainer, and usage billing inputs
- Issues, risks, decisions, changes, approvals, and lessons learned
- Customer collaboration, deliverables, acceptance, and project portals
- Project dashboards, earned value hooks, profitability, and utilization

## Authoritative Entities

Project, Program, Portfolio, Phase, Milestone, Project Task, Assignment, Project Budget, Project Risk, Change Request, Deliverable, and Acceptance.

## Boundaries

Workforce owns workers and skills. Scheduling Engine provides calendar behavior. Finance owns accounting and invoices. Commerce or Service may originate customer commitments. Projects owns coordinated delivery scope and progress.

## Quality Requirements

- Effective-dated baselines and change history
- Permissioned customer and contractor collaboration
- Time and cost reconciliation
- Dependency and critical-path integrity
- Mobile and offline field updates
- Clear distinction between planned, forecast, committed, and actual values
