# Competitor Benchmark Scorecard

## Purpose

Provide a repeatable format for comparing one workflow across the platform, direct competitors, specialist products, and manual alternatives.

## Benchmark Header

- Workflow
- Persona
- Industry and jurisdiction
- Date
- Product, edition, and version
- Evaluator
- Source and evidence quality
- Test data and environment
- Known limitations

## Measures

### Outcome

- Task completed
- Correct result
- Recovery from mistakes
- Business consequence

### Effort

- Time
- Steps
- Clicks or taps
- Typing
- Scans
- Context switches
- Training required

### Quality

- Error rate
- Correction rate
- Discoverability
- Clarity
- Trust
- Accessibility
- Mobile usability
- Offline behavior

### Platform Depth

- Configuration
- Custom fields
- Permissions
- Audit
- Reporting
- APIs
- Integrations
- Migration
- Reliability
- Support
- Price transparency

## Workflow Score

Score each dimension 1–5 with confidence and evidence. Record raw measurements separately. Do not average incomparable dimensions without declared weighting.

## Comparative Narrative

Explain:

- Where the competitor is materially better
- Where the platform is materially better
- Where differences are irrelevant to the target user
- Which advantage depends on implementation or services
- Which gap should be closed, deferred, partnered, or intentionally ignored

## Initial Scorecards

- Store setup and product import
- Cash sale
- Mixed-tender sale
- Return and refund
- Register close and deposit
- Inventory count
- Purchase order and receiving
- User and role setup
- Custom field creation
- Dashboard creation
- Integration setup
- Offline operation

## Evidence Rules

Prefer direct product testing, official documentation, customer implementation evidence, and reproducible screenshots. Separate vendor claim, observed behavior, inference, and recommendation.
