# Execution Prompt Pack (fifth audit)

Self-contained prompts for remediation waves and upcoming implementation PRs. Each is usable by any capable agent in a fresh session with no hidden memory. **Shared preamble for every prompt** (prepend verbatim):

> Work in the platform-design-authority repository. Read `AGENTS.md` completely first and obey its authority order, naming, and prohibited behaviors. Base from current `main`; one issue, one branch, one worktree, one PR. Normative finding definitions and closure tests: `evidence/audit/fable5-whole-project-findings.yaml`; plan and wave assignments: `docs/reviews/FABLE5_FIFTH_AUDIT_REMEDIATION_PLAN_V1.md`. Never edit `docs/reviews/FABLE5_FIFTH_AUDIT_V1.md` (immutable evidence), never hand-edit generated registries or committed migrations, never mark a finding closed — status changes go in the remediation plan and risk register with evidence pointers. Before PR: run `bun run check-types`, `bun run test`, `bun run check`, `python scripts/validate_docs.py`, `python scripts/generate_registries.py --check`, plus the prompt's specific gates. Stop and record the missing decision if any step requires authority you do not have. Merge only after green exact-head CI and independent exact-head concurrence recorded on the PR; after merge, verify the exact main SHA re-runs the required gates green and record it. Rollback: every PR here must be a clean single revert.
